Componentes Na versão 1 a Move.CSS possui componentes básicos para estilização de inputs para formulários.
Botões Os botões da Move.CSS possuem variações nas 4 cores definidas pela biblioteca
<button class="mv-button default">padrão</button> <button class="mv-button success">sucesso</button> <button class="mv-button alert">alerta</button> <button class="mv-button error">erro</button>
A Move.CSS também possui classes para botões de ação secundários
<button class="mv-button default-secondary">padrão</button> <button class="mv-button success-secondary">sucesso</button> <button class="mv-button alert-secondary">alerta</button> <button class="mv-button error-secondary">erro</button>
Além de botões para ações genéricas, a Move.CSS possui botões do tipo submit usados para envio de formulários, estes com classes para feedback em caso de sucesso e erro
<button class="mv-button submit" type="submit" id="ok">enviar sucesso</button> <button class="mv-button submit" type="submit" id="error">enviar erro</button>
Para exibir os feedbacks de sucesso e erro é necessário o uso do Jquery para alternar entre as classes submitting e submitedError ou submitedSuccess após a resposta da submissão
//Chamar função de submitting ao clicar no botão desejado $("#enviar").trigger('submitting'); //Chamar função de sucesso ou erro após a resposta da submissão $("#enviar").trigger('submitedSuccess'); $("#enviar").trigger('submitedError');
Inputs Os componentes de input da Move.CSS devem ser filhos de um elemento div com a classe mv-input-container, este por sua vez possui a largura máxima do elemento pai, portanto é possível defini-lo como filho de uma coluna bootstrap, por exemplo.
<div class="col-md-3"> <div class="mv-input-container"> </div> </div>
Os inputs tipo: text, password, email, checkbox e radio devem estar dentro de uma classe mv-input-container.
Input text Os componentes de input do tipo text podem receber todas as propriedades da tag input, e devem ser seguidos por um elemento div com a classe input-border, e opcionalmente por um elemento label com a classe mv-label e demais propriedades da tag label.
<div class="col-md-3"> <div class="mv-input-container"> <input type="text" class="mv-input" id="nome" placeholder="Ex: José"/> <div class="input-border"></div> <label class="mv-label" for="nome">Nome</label> </div> </div>
Input password Os components de input do tipo password contam um password reveal, para o usá-lo é necessário adicionar ao projeto o arquivo SVG disponível na pasta assets da biblioteca e referenciá-lo em uma tag object com a classe mv-password-reveal.
<div class="col-md-3"> <div class="mv-input-container"> <input type="password" class="mv-input" data-reveal="true" id="pass"/> <span class="input-border"></span> <label class="mv-label" for="pass">Senha</label> </div> </div>
Além do password reveal, os componentes de input tipo password também podem possuir um elemento de de validação da força da senha com as classes mv-password-strength + a classe referente a força da senha inserida.
<div class="mv-input-container"> <input type="password" class="mv-input" data-reveal="true" id="pass"/> <span class="input-border"></span> <span class="mv-password-strength strong">forte</span> <label class="mv-label" for="pass">Senha</label> </div> <div class="mv-input-container"> <input type="password" class="mv-input" data-reveal="true" id="pass2"/> <span class="input-border"></span> <span class="mv-password-strength mid">média</span> <label class="mv-label" for="pass">Senha</label> </div> <div class="mv-input-container"> <input type="password" class="mv-input" data-reveal="true" id="pass3"/> <span class="input-border"></span> <span class="mv-password-strength weak">fraca</span> <label class="mv-label" for="pass">Senha</label> </div>

Input checkbox O componente de input tipo checkbox da Move.CSS deve possuir um id e ser seguido por um elemento label que deve referenciar o id do input na propriedade for.
<div class="mv-input-container"> <input type="checkbox" class="mv-input" id="cb" name="cb"/> <label class="mv-label" data-label="checkbox" for="cb">checkbox</label> </div>
Input radio Semelhante ao input tipo checkbox, o input radio também deve ser seguido por um elemento label.
<div class="mv-input-container"> <input type="radio" class="mv-input" id="radio" name="radio"/> <label class="mv-label" data-label="radio" for="radio">radio</label> </div>
